Djikstra's Algorithm Visualiser

Welcome to my Pathfinding Algorithm Visualiser!

1/4

This is the first iteration of my pathfinding visualiser.
You can press "Next" to see a breif tutorial on how to use the visualiser.
Or press "Skip Tutorial" to go straight to using it.

1. Start and end points

2/4

In order for the algorithm to find a path, you first need to set a start node and an end node.

To set the start node, press the green "Set Start" button at the top of the page then click on the node on the grid that you would like to set as your starting point.

To set the end node, press the red "Set End" button at the top of the page then click on the node on the grid that you would like to set as your end point.

2. Walls

3/4

You can place walls over nodes. We cannot travel through a wall, so when you place a wall over a node the algorithm has to find a way around it.

To place walls on the grid, press the black "Set Walls" button at the top of the page. You can then click and drag your cursor across the grid to place walls.

If you make a mistake, or just want to remove some walls, press the grey "Erase Walls" button at the top of the page. Erasing walls also works by clicking and dragging the cursor.

3. Find Path

4/4

Finally, with all of your walls in place, and your starting and ending nodes set. Simply press the blue "Find Path" button to run the algorithm and find your shortest path.

The program uses 'Djikstra's Algorithm' to find the shortest path between your start and end nodes. I intend to add more algorithms to the program over time.

When your path has been found press the "Reset" button to clear the grid and start again!
